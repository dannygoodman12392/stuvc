# SYSTEM_MAP.md — Stu VC Operating System

> Phase 0 discovery map. Read-only audit; no code changed. Every claim cites file + line.
> Generated by a 6-agent read-only audit of the live codebase.

## 1. What this system is

A single-tenant-in-practice (user_id=1), multi-tenant-capable VC operating system:
React (Vite) SPA + Node/Express API + **SQLite** (`server/superior-os.db`), deployed on
Railway. Claude (Anthropic) powers scoring/extraction. External mirrors: Airtable (CRM),
Notion (Danny OS system of record).

---

## 2. The four products — where each lives

### a. Daily Briefing — `Working`
Ingests newsletters → ranked VC news/insights.
- `server/services/newsletter.js` — RSS auto-discovery (`discoverFeedUrl`, L362–400), email-sender pull via Gmail IMAP (`fetchEmailSenders`, L424–465), Claude extraction, relevance scoring, dedupe by `message_id`.
- `server/routes/newsletter.js` — `/brief`, `/sync`, `/status`, sources CRUD.
- `client/src/pages/Brief.jsx` — rolling multi-day feed, 3-tier grouping.
- `server/index.js` L201–225 — 6 AM CT cron.
- Tables: `newsletter_items`, `newsletter_sources`.

### b. Sourcing — `Working`
Chicago/IL pre-seed + stealth builders.
- `server/pipeline/sourcing-engine.js` — Exa + GitHub discovery, **hard tie gate** (`VALID_TIE_TYPES`, L556; enforced L1501), caliber model (`computeCaliber`/`reconcileCaliber`, L487), `verifyPedigree` (L309), founder gate.
- `server/pipeline/taste.js` — learning loop (affinity from approve/star/dismiss).
- `server/routes/sourcing.js` — queue, approve/star/dismiss, stats.
- Table: `sourced_founders` (34 cols).

### c. Talent — `Working`
Best-fit candidates for founders' open roles.
- `server/pipeline/match-engine.js` — **function gate** (`functionFits`/`resolveRoleFunction`, L64–79; gate before scoring L228), heuristic match (band/stack/caliber/leap/domain/location).
- `server/pipeline/talent-engine.js` — archetype sourcing + scoring (`ARCHETYPES`, L558–631), tier-1 gate, `verifyPedigree`.
- `server/routes/talent/*.js`. Tables: `talent_portfolio_companies`, `talent_roles`, `talent_candidates`, `talent_matches`.

### d. Founder Assessment — `Partial` ⚠️
Multi-agent research + evaluation.
- `server/agents/prompts.js` — 6 agents (Team/Product/Market/Bear/Synthesis/Steward-Operator), deterministic rubrics.
- `server/routes/assessments.js` — orchestration, deterministic pillar scores, budget-aware context assembly.
- `server/agents/verify.js` — quote verification (verbatim/paraphrased/unverified).
- `client/src/pages/Assess.jsx`, `AssessmentDetail.jsx`.
- **Broken input path:** decks are read client-side via `FileReader.readAsText()` (`Assess.jsx` L71) — corrupts PDF/PPTX binary. `server/services/pdf-extractor.js` exists but is **never imported/called**. PPTX has no extractor at all.

## 3. Data stores (catalog + drift)

| Store | Role | Reads | Writes | Drift risk |
|---|---|---|---|---|
| **SQLite** `superior-os.db` (26 tables) | **Source of truth** | all routes | routes + background jobs | — |
| **Airtable** `appfE9DVrSUOrkkpu` | CRM mirror | import on startup (`airtable-import.js`) + `/founders/sync` | one-way push on stage change (`airtable-sync.js`), fire-and-forget | **HIGH** — push failures silent; startup re-import can overwrite unsynced Stu changes; duplicate risk if record-id never saved |
| **Notion** (Danny OS) | Memos/founders SoR | none (write-only) | `notion-sync.js`, `notion-assessment-sync.js`, fire-and-forget | **MEDIUM** — protects Strider Stage/Notes/Conviction on update, but overwrites name/company/stage; never read back |
| **Granola** | meeting transcripts | — | **none** — `call_logs` is manual paste only (`routes/calls.js`); no Granola API integration | LOW (not integrated) |
| **Entity Filings** (SEC/IL SOS) | upstream signal | sourcing engine | `filings-source.js` | LOW |

**Founder data is canonical in SQLite but mirrored one-way to BOTH Airtable and Notion with no transaction, no retry, no read-back.** This is the central drift surface.

- Crons (node-cron): newsletter 6 AM CT (always); sourcing/talent/filings (gated by `PIPELINE_ENABLED=true`).

## 7. Headline issues Phase 1 will rank (preview)
- **P0 — Assessment deck corruption.** PDF/PPTX read as text → garbage fed to agents → confident scores on broken input, no error. The single biggest trust hole.
- **P0 — Silent failures everywhere.** Newsletter sync and all Airtable/Notion syncs report success while failing. Violates "fail loudly."
- **P1 — Sourcing approve is non-transactional** → duplicate/orphan founders.
- **P1 — Promotion drops metadata** (pedigree/evidence/red_flags not copied to `founders`).
- **P1 — No per-attribute sourcing/citation** surfaced; evidence captured but not shown.
- **P2 — Talent ignores must-haves/nice-to-haves; domain match is substring-only.**
- **P2 — PII**: transcripts/assessment content unencrypted at rest (inherent to SQLite; flag).
